# 배열과 연결리스트의 차이

## 핵심 요약
배열은 연속된 메모리를 사용해 인덱스 접근이 O(1)로 빠르다.
연결리스트는 노드가 포인터로 연결되어 삽입·삭제가 유연하지만 탐색은 O(n)이다.

---

## 상세 설명

### 배열 (Array)
- 메모리에 **연속적**으로 저장
- 인덱스로 임의 접근 → O(1)
- 크기가 고정 (정적 배열) 또는 재할당 발생 (동적 배열)
- 중간 삽입/삭제 시 요소를 밀어야 함 → O(n)

### 연결리스트 (Linked List)
- 각 노드가 `데이터 + 다음 노드 포인터` 로 구성
- 메모리 비연속 — 포인터로 연결
- 특정 위치 탐색 → O(n) (처음부터 순회)
- 삽입/삭제: 포인터만 변경 → O(1) (위치를 알고 있을 때)

### 비교 요약

| 연산 | 배열 | 연결리스트 |
|------|------|-----------|
| 임의 접근 | O(1) | O(n) |
| 삽입 (끝) | O(1) | O(1) |
| 삽입 (중간) | O(n) | O(1)* |
| 삭제 (중간) | O(n) | O(1)* |
| 메모리 | 연속 | 비연속 |

*위치를 이미 알고 있을 때

---

## 키포인트
- 읽기가 많으면 배열, 삽입/삭제가 많으면 연결리스트
- 캐시 지역성: 배열이 유리 (연속 메모리)
- 연결리스트는 추가 포인터 메모리 오버헤드 존재
