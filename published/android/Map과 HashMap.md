> Map은 Key-Value 쌍으로 데이터를 저장하는 인터페이스이며, HashMap은 해시 함수로 O(1) 접근을 구현한 대표 구현체다. 충돌은 연결리스트→트리 전환으로 최악의 경우에도 O(log n)을 보장한다.

## 정의
- Map: 키(Key)와 값(Value)의 쌍으로 데이터를 저장하는 인터페이스. Key는 중복 불가, Value는 중복 허용
- HashMap: Map 인터페이스의 구현체. 해시 함수로 버킷 인덱스를 계산해 평균 O(1) 접근 성능을 제공
- 해시 함수: 임의 크기의 키를 고정된 크기의 정수(해시값)로 변환하는 함수. h(k) = v
- 해시 충돌: 서로 다른 키가 동일한 해시값을 가지는 상황. 불가피하게 발생하며 연결리스트·트리로 처리
- 버킷(Bucket): HashMap 내부 배열의 각 슬롯. 기본 16개로 시작하며 항상 2의 거듭제곱 유지
- 로드 팩터(Load Factor): 버킷 대비 저장된 데이터 비율. 기본값 0.75 초과 시 리사이징 발생
- 리사이징(Resizing): 용량이 초과될 때 배열을 2배로 늘리고 모든 데이터를 재배치하는 과정

## 핵심 포인트
- HashMap 내부 구조는 배열 + 연결리스트. 충돌이 8개 이상 누적되면 연결리스트를 Red-Black 트리로 변환해 O(n) → O(log n)으로 개선한다
- 인덱스 계산식 `(n-1) & hash`: 배열 크기(n)가 2의 거듭제곱일 때만 mod 연산과 동일하게 동작하기 때문에 크기는 항상 2의 거듭제곱을 유지한다
- HashMap은 데이터 순서를 보장하지 않는다. 삽입 순서 유지 → LinkedHashMap, 키 정렬 → TreeMap을 사용한다
- 좋은 해시 함수는 데이터를 전체 버킷에 균등하게 분산시켜 충돌을 최소화한다
- 데이터 크기를 예측할 수 있다면 초기 용량을 직접 지정해 리사이징 비용을 줄일 수 있다
