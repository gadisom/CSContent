> 탐색은 원하는 값을 찾는 것에서 끝나지 않고, 상태 공간을 어떤 순서로 줄여 나갈지 설계하는 문제다.

## 정의
- 탐색 문제는 단순 선형 탐색, 정렬 후 이진 탐색, 그래프 탐색, 백트래킹처럼 여러 방식으로 풀 수 있다. 핵심은 데이터 구조와 조건을 보고 어떤 탐색 틀을 적용할지 고르는 일이다.
- 입력 규모가 작으면 완전 탐색이 가능하지만, 조건이 커지면 탐색 공간을 줄이는 전략이 필요하다. 이때 정렬, 방문 체크, 가지치기, 메모이제이션 같은 도구가 함께 쓰인다.
- 면접에서 탐색을 설명할 때는 특정 알고리즘 이름만 말하지 말고, 왜 그 방식이 문제의 구조와 맞는지까지 연결하는 것이 좋다.

## 핵심 포인트
- 탐색은 문제 구조에 맞는 전략 선택이 중요하다.
- 탐색 공간 축소는 성능 개선의 핵심이다.
- 정렬, 방문 배열, 해시맵은 탐색 보조 도구로 자주 쓰인다.

## 면접 질문
- 탐색 문제를 풀 때 어떤 관점으로 접근하나요?
- 완전 탐색과 효율적인 탐색의 차이는 무엇인가요?

## 확인 문제
- 탐색은 항상 값 하나를 찾는 문제에만 쓰일까?
- 탐색 공간을 줄이려면 조건을 활용해야 할까?
