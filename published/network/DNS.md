> DNS는 사람이 읽기 쉬운 도메인 이름을 IP 주소로 변환하는 시스템이다. 브라우저 캐시부터 재귀 질의까지 단계적으로 조회된다.

## 정의
- 사용자가 도메인을 입력하면 브라우저나 운영체제는 먼저 로컬 캐시를 확인한다. 없으면 DNS 서버에 질의해 해당 도메인의 IP를 찾아간다.
- 이 과정은 루트 네임서버, TLD 서버, 권한 있는 네임서버를 거치는 단계적 조회로 이뤄질 수 있다. 캐시가 잘 동작하면 실제 전체 과정을 매번 반복하지는 않는다.
- 면접에서는 DNS를 단순 전화번호부 비유에서 끝내지 말고, 왜 웹 요청 전에 DNS 조회가 먼저 필요한지까지 이어서 설명하면 좋다.

## 핵심 포인트
- DNS는 이름을 주소로 바꾸는 시스템이다.
- 캐시는 DNS 성능에 큰 영향을 준다.
- 웹 요청 흐름의 초반 단계에서 핵심 역할을 한다.
