> 쿠키는 클라이언트 측에 저장되는 정보이고, 세션은 서버 측 상태 관리 개념이다. 둘은 대체 관계보다 함께 쓰이는 경우가 많다.

## 정의
- HTTP는 기본적으로 상태를 기억하지 않기 때문에 로그인 같은 기능을 위해 상태 관리 장치가 필요하다. 쿠키는 브라우저에 저장되어 요청마다 함께 전송될 수 있는 데이터다.
- 세션은 서버가 사용자 상태를 관리하는 방식이다. 보통 브라우저에는 세션 자체가 아니라 세션 식별자만 쿠키로 저장하고, 실제 사용자 정보는 서버에 둔다.
- 면접에서는 쿠키와 세션을 완전히 별개라고만 설명하기보다, 세션 기반 인증에서 둘이 어떻게 함께 동작하는지 설명하는 것이 중요하다.

## 핵심 포인트
- 쿠키는 저장 위치, 세션은 상태 관리 방식에 가깝다.
- 세션 ID를 쿠키에 담아 함께 쓰는 경우가 많다.
- 보안과 확장성을 고려해 저장 내용을 설계해야 한다.

## 면접 질문
- 쿠키와 세션의 차이를 설명해보세요.
- 세션 기반 로그인은 어떻게 동작하나요?

## 확인 문제
- 세션은 항상 브라우저에 직접 저장될까?
- 쿠키는 클라이언트 측 저장소 역할을 할 수 있을까?
