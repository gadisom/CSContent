> 브라우저 요청은 DNS 조회, TCP 연결, HTTP 요청, 서버 처리, 응답 렌더링 같은 여러 계층 단계를 거친다.

## 정의
- 사용자가 URL을 입력하면 브라우저는 먼저 캐시와 DNS를 통해 서버 주소를 확인한다. 그다음 TCP 또는 TLS 연결을 준비하고 HTTP 요청을 전송한다.
- 서버는 요청을 처리해 응답을 보내고, 브라우저는 HTML을 파싱하며 추가 리소스를 요청하고 렌더링을 진행한다. 이 과정은 네트워크와 브라우저 내부 동작이 함께 얽혀 있다.
- 면접에서는 이 흐름을 계층별로 설명하면 좋다. DNS, TCP handshake, TLS, HTTP, 렌더링을 큰 단계로 나눠 말하면 정리된 인상을 준다.

## 핵심 포인트
- 웹 요청은 하나의 동작처럼 보여도 여러 단계가 이어진 결과다.
- DNS, 연결 수립, 애플리케이션 요청, 렌더링이 핵심 흐름이다.
- 네트워크와 브라우저 동작을 함께 설명할 수 있어야 한다.
