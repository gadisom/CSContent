> GET은 주로 조회, POST는 서버 상태 변경을 위한 요청에 사용된다. URL 노출 여부만이 아니라 의미와 멱등성 차이가 중요하다.

## 정의
- GET은 리소스를 조회하는 의미를 갖고, 동일 요청을 여러 번 보내도 서버 상태를 바꾸지 않는 것이 이상적이다. 캐싱과 북마크, 링크 공유와도 잘 맞는다.
- POST는 서버에 새로운 처리나 상태 변경을 요청할 때 많이 사용한다. 요청 본문에 데이터를 담아 보내는 경우가 일반적이지만, 본문 사용 여부가 본질은 아니다.
- 면접에서는 GET은 안전하고 POST는 안전하지 않다처럼 단순화하기보다, 메서드의 의도와 멱등성 차이를 설명하는 것이 더 정확하다.

## 핵심 포인트
- 메서드 구분의 핵심은 의도와 의미다.
- GET은 조회, POST는 처리 요청에 가깝다.
- URL에 보인다고 해서 보안성이 자동으로 결정되지는 않는다.
