> SQL Injection은 사용자 입력이 쿼리 구조를 바꿔 버리면서 발생하는 공격이다. 입력 검증보다 중요한 기본 대응은 쿼리와 데이터를 분리하는 것이다.

## 정의
- SQL Injection은 사용자 입력이 SQL 일부로 해석되도록 만들 때 발생한다. 로그인 우회, 데이터 노출, 데이터 변조 같은 심각한 문제로 이어질 수 있다.
- 가장 기본적이고 중요한 대응은 Prepared Statement처럼 파라미터 바인딩을 사용하는 것이다. 여기에 입력 검증과 최소 권한 원칙을 더하면 방어 수준이 높아진다.
- 면접에서는 단순히 특수문자를 막는다고 끝내지 말고, 쿼리 구조와 데이터를 분리하는 설계가 핵심이라는 점을 분명히 말하는 것이 좋다.

## 핵심 포인트
- SQL Injection은 쿼리 구조 오염 문제다.
- Prepared Statement가 가장 기본적인 대응책이다.
- 입력 검증과 DB 권한 최소화도 함께 필요하다.
