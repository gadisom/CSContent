> 가상 메모리는 프로세스마다 큰 연속 주소 공간이 있는 것처럼 보이게 하고, 실제 메모리에는 필요한 페이지만 올려 사용한다.

## 정의
- 가상 메모리는 물리 메모리보다 더 큰 주소 공간을 사용하는 것처럼 보이게 만드는 추상화다. 실제로는 현재 필요한 페이지들만 메모리에 올리고 나머지는 디스크에 둔다.
- 프로세스가 메모리에 없는 페이지를 참조하면 페이지 폴트가 발생하고, 운영체제는 필요한 페이지를 적재한다. 메모리가 부족하면 어떤 페이지를 내릴지 결정하는 페이지 교체 알고리즘이 필요하다.
- 면접에서는 요구 페이징, 페이지 폴트, 지역성, 교체 알고리즘이 서로 연결된 개념이라는 흐름으로 설명하면 이해가 잘 된다.

## 핵심 포인트
- 가상 메모리는 큰 주소 공간과 효율적 메모리 사용을 동시에 노린다.
- 요구 페이징은 필요한 순간에만 페이지를 적재한다.
- 페이지 교체는 성능에 직접 영향을 준다.

## 면접 질문
- 가상 메모리를 설명해보세요.
- 페이지 폴트가 발생하면 어떤 일이 일어나나요?

## 확인 문제
- 가상 메모리는 모든 페이지를 항상 메모리에 올려둘까?
- 페이지 교체 알고리즘은 왜 필요할까?
