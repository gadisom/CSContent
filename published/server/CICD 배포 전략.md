> 블루그린은 두 환경을 동시에 유지하며 로드밸런서만 전환해 즉각 롤백이 가능하고, 카나리는 일부 트래픽에 점진적으로 배포해 위험을 최소화한다.

## 정의
- 블루그린 배포: 현재 운영(Blue)과 신규(Green) 두 환경을 동시에 유지. 로드밸런서만 전환하면 되므로 롤백이 즉각적이다.
- 카나리 배포: 신규 버전을 소수 사용자에게 점진적으로 오픈해 모니터링한 뒤 전체 전환. AB 테스트에도 활용된다.
- Nginx 로드밸런싱: upstream 블록으로 라운드 로빈(기본), weighted round robin, least connection, IP Hash 등을 지원한다.

## 핵심 포인트
- 블루그린은 롤백이 빠르지만 두 배의 인프라 비용이 필요하다.
- 카나리는 점진적 배포로 위험을 최소화하고, 완전 전환 후 이전 인스턴스를 삭제해 자원을 효율화한다.
- Nginx 기본 로드밸런싱은 라운드 로빈으로 순차 분배한다.
- IP Hash는 동일 클라이언트가 항상 같은 서버로 연결되어 세션 유지가 필요한 경우에 적합하다.
