> RabbitMQ는 AMQP 기반 신뢰성 전달에 강하고, Kafka는 로그 기반 고성능 스트리밍에 강하다. 메시지 저장 방식과 재처리 가능 여부가 핵심 차이다.

## 정의
- 메시지큐: 프로듀서와 컨슈머 사이에서 메시지를 비동기로 전달하는 미들웨어
- RabbitMQ: AMQP 프로토콜 기반. Exchange(수신)→Queue(저장)→Consumer 구조. 메시지를 주로 메모리에 저장해 처리가 빠르지만 유실 위험이 있음. 컨슈머가 ACK를 보내야 큐에서 제거됨
- Kafka: TCP 기반 자체 프로토콜. 메시지를 디스크에 로그 형태로 저장. 컨슈머가 받아가도 삭제되지 않으며 오프셋으로 위치를 추적해 재처리가 가능함

## 핵심 포인트
- RabbitMQ는 ACK 기반 신뢰성 전달을 보장해 트랜잭션성 작업에 적합하다.
- Kafka는 배치 처리와 클러스터링에 유리해 대용량 스트리밍에 적합하다.
- Kafka의 오프셋 기반 구조 덕분에 컨슈머 장애 후 재전송·특정 메시지 재처리가 가능하다.
- RabbitMQ는 큐, Kafka는 토픽으로 메시지 그룹을 구분한다.
- RabbitMQ는 메시지가 여러 큐나 다양한 컨슈머로 유연하게 라우팅 가능해 다양한 메시지 처리에 유리하다.
