> Docker는 애플리케이션을 컨테이너로 격리해 실행 환경을 통일한다. 이미지는 컨테이너 생성을 위한 불변 템플릿이며, 커널을 공유하므로 VM보다 가볍다.

## 정의
- 컨테이너: 호스트 OS 커널을 공유하면서 격리된 실행 환경을 제공하는 단위. VM과 달리 자체 OS 커널이 없어 가볍고 빠르게 기동된다.
- 이미지: 컨테이너를 만들기 위한 불변 템플릿(금형). 레이어 구조로 재사용성이 높다. Docker Hub에서 다운로드 가능.
- Docker Engine: 컨테이너를 생성·실행하는 데몬. 리눅스 기반 소프트웨어로, 이미지로부터 컨테이너를 만들어 격리된 환경을 제공한다.

## 핵심 포인트
- 컨테이너는 커널을 공유하므로 VM보다 가볍고 기동이 빠르다.
- 이미지는 불변(immutable). 컨테이너는 이미지 위에 쓰기 레이어를 추가한 것이다.
- Docker Hub에서 공식 이미지를 다운받아 바로 사용할 수 있다.
- 컨테이너마다 환경이 격리되어 "내 컴퓨터에서는 됩니다" 문제를 해결한다.
