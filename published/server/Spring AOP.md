> Spring AOP는 JDK 동적 프록시(인터페이스 기반)와 CGLIB(클래스 기반)으로 프록시를 생성한다. 빈 후처리기가 컴포넌트 스캔된 빈에도 자동으로 프록시를 적용한다.

## 정의
- JDK 동적 프록시: 인터페이스 기반 프록시 생성. InvocationHandler로 부가 로직을 추가한다.
- CGLIB: 클래스 기반 프록시 생성. MethodInterceptor 구현. Spring 4.0부터 objenesis 라이브러리로 기본 생성자 없이도 객체 생성이 가능해 생성자 2회 호출 문제가 해결됐다.
- 포인트컷: 부가 기능을 어디에 적용할지 결정하는 필터링 로직. AspectJ 표현식이 실무에서 가장 많이 쓰인다.
- 어드바이스: 프록시가 호출하는 부가 기능 로직. @Around, @Before, @AfterReturning, @AfterThrowing, @After 종류가 있다.
- 어드바이저: 포인트컷 1개 + 어드바이스 1개를 가지는 단위.
- 빈 후처리기: 빈 저장소 등록 전 빈을 조작하거나 프록시로 교체하는 컴포넌트. 컴포넌트 스캔 대상에도 자동으로 AOP 프록시를 적용한다.

## 핵심 포인트
- Spring AOP는 기본적으로 CGLIB을 사용해 구체 클래스 기반 프록시를 생성한다.
- 하나의 target에 여러 AOP가 적용되어도 프록시는 하나만 생성된다. proxyFactory.addAdvisor()로 여러 어드바이저를 하나의 프록시에 등록한다.
- @Aspect로 선언된 어드바이저를 빈 후처리기가 스캔해 포인트컷 조건에 따라 프록시 적용 여부를 자동 판단한다.
- ProxyFactory만 사용하면 컴포넌트 스캔 빈에는 자동 프록시 적용이 불가 → 빈 후처리기가 이 문제를 해결한다.
