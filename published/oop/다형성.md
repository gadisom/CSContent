> Swift 다형성은 클래스 오버라이딩(vtable)과 프로토콜 채택(witness table) 두 축으로 구현되며, 오버로딩은 컴파일 타임에 해결된다.

## 정의
- 다형성(Polymorphism): 동일한 인터페이스로 다양한 타입의 객체를 다룰 수 있는 OOP 원칙
- 오버라이딩(Overriding): 자식 클래스가 부모 클래스의 메서드를 재정의하는 것. Swift에서는 override 키워드 필수
- 오버로딩(Overloading): 같은 이름이지만 파라미터 타입·개수가 다른 메서드를 여러 개 정의하는 것. 컴파일 타임에 해결
- vtable(Virtual Dispatch Table): class 메서드 오버라이딩을 위해 런타임에 실제 구현을 찾는 테이블. dynamic dispatch의 구현체
- witness table: Protocol 채택 타입별로 각 요구사항의 실제 구현 주소를 저장하는 테이블. Protocol 다형성의 구현체
- 타입 캐스팅: as?, as!, is 연산자로 부모→자식 또는 Protocol 타입으로의 변환을 수행

## 핵심 포인트
- class는 기본적으로 dynamic dispatch를 사용한다. final을 붙이거나 @objc 없이 private으로 선언하면 static dispatch로 최적화된다.
- Protocol 다형성은 witness table로 구현되며, class 상속과 독립적으로 동작해 struct·enum에도 적용 가능하다.
- 오버라이딩 시 super.method()로 부모 구현을 호출할 수 있다. 생성자(init)는 designated·convenience·required 규칙을 따른다.
- as?는 다운캐스팅 실패 시 nil을 반환하고, as!는 런타임 크래시를 유발한다. 가능하면 as? + guard/if let을 사용한다.
- 오버로딩은 Swift에서 광범위하게 사용된다. 같은 이름의 연산자·함수를 타입별로 구현하는 제네릭 설계에도 활용된다.
