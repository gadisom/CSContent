#### OX | [679]
오버라이딩(Overriding)과 오버로딩(Overloading)은 모두 다형성을 구현하는 방법이다.
> O
> 오버라이딩은 런타임 다형성(동적 바인딩), 오버로딩은 컴파일 타임 다형성(정적 바인딩)이다. 둘 다 같은 이름으로 다른 동작을 표현하는 다형성의 형태다.

#### OX | [680]
부모 타입의 참조 변수로 자식 타입의 인스턴스를 가리킬 수 있다.
> O
> 업캐스팅(upcasting)이라 부른다. 부모 타입 변수에 자식 객체를 담아 공통 인터페이스로 다루는 것이 런타임 다형성의 핵심이다.

#### 빈칸 | [681]
자식 클래스에서 부모 클래스의 메서드를 같은 시그니처로 재정의하는 것을 ___ 이라 한다.
> 오버라이딩
> 오버라이딩된 메서드는 런타임에 실제 객체 타입을 기준으로 호출된다(동적 바인딩). 이것이 런타임 다형성의 동작 원리다.

#### 빈칸 | [682]
같은 이름이지만 파라미터 타입이나 개수가 다른 메서드를 여러 개 정의하는 것을 ___ 이라 한다.
> 오버로딩
> 오버로딩은 컴파일 타임에 어떤 메서드를 호출할지 결정된다. 반환 타입만 다른 경우는 오버로딩으로 인정하지 않는 언어가 많다.

#### 객관식 | [683]
런타임 다형성(동적 바인딩)이 일어나는 조건으로 옳은 것은?
1. 메서드 이름과 파라미터가 동일하면 자동으로 발생한다.
2. ✅ 부모 타입의 참조로 자식 객체를 가리키고 오버라이딩된 메서드를 호출할 때 발생한다.
3. static 메서드를 오버라이딩할 때 발생한다.
4. 생성자를 오버로딩할 때 발생한다.
> 동적 바인딩은 부모 타입 참조 → 자식 객체 → 오버라이딩 메서드 조합에서 발생한다. static 메서드나 생성자는 컴파일 타임에 결정되어 동적 바인딩 대상이 아니다.

#### 객관식 | [684]
다형성을 활용하는 설계 사례로 가장 적절한 것은?
1. 모든 클래스 필드를 public으로 선언해 어디서나 접근 가능하게 한다.
2. 각 동물 클래스마다 makeSound()를 호출하는 코드를 따로 작성한다.
3. ✅ Animal 타입의 리스트에 Dog·Cat·Bird 객체를 담고 루프에서 makeSound()를 일괄 호출한다.
4. 자식 클래스마다 서로 다른 이름의 메서드를 정의해 충돌을 방지한다.
> 공통 부모 타입으로 다양한 객체를 묶어 동일한 메시지(makeSound)를 보내는 것이 다형성의 핵심 활용이다. 새 동물 클래스를 추가해도 호출 코드를 수정할 필요가 없어 OCP(개방-폐쇄 원칙)를 만족한다.
