#### OX | [631]
Spring AOP는 기본적으로 JDK 동적 프록시를 사용해 인터페이스 기반 프록시를 생성한다.
> X
> Spring AOP는 기본적으로 CGLIB을 사용해 구체 클래스 기반 프록시를 생성한다. JDK 동적 프록시는 인터페이스가 있을 때 사용하는 방식이다.

#### OX | [632]
하나의 target에 어드바이저 3개를 적용하면 프록시가 3개 생성된다.
> X
> proxyFactory.addAdvisor()로 여러 어드바이저를 하나의 프록시에 등록할 수 있다. 따라서 어드바이저가 몇 개든 프록시는 1개만 생성된다.

#### 빈칸 | [633]
CGLIB에서 기본 생성자 2회 호출 문제를 Spring 4.0부터 해결해준 라이브러리는 ___ 이다.
> objenesis
> objenesis는 기본 생성자 없이도 객체를 생성할 수 있게 해준다. 이로 인해 CGLIB 프록시 생성 시 원본 객체 생성자 호출 문제가 해결됐다.

#### 빈칸 | [634]
부가 기능을 어디에 적용할지 결정하는 AOP의 필터링 로직을 ___ 이라고 한다.
> 포인트컷
> 포인트컷은 어드바이스를 적용할 메서드나 위치를 지정하는 조건이다. 실무에서는 AspectJ 표현식이 가장 많이 쓰인다.

#### 객관식 | [635]
빈 후처리기가 해결하는 문제는?
1. 트랜잭션 동기화
2. 커넥션 풀 관리
3. ✅ 컴포넌트 스캔으로 등록된 빈에 자동으로 AOP 프록시를 적용
4. 예외 처리 흐름 관리
> ProxyFactory만으로는 컴포넌트 스캔 빈에 자동 프록시를 적용할 수 없다. 빈 후처리기가 빈 등록 전에 어드바이저 조건을 확인하고 프록시로 교체해 이 문제를 해결한다.

#### 객관식 | [636]
JDK 동적 프록시와 CGLIB의 차이로 올바른 것은?
1. JDK 동적 프록시는 클래스 기반, CGLIB은 인터페이스 기반이다
2. ✅ JDK 동적 프록시는 인터페이스 기반, CGLIB은 클래스 기반으로 프록시를 생성한다
3. 둘 다 인터페이스가 필요하다
4. Spring AOP는 JDK 동적 프록시만 사용한다
> JDK 동적 프록시는 InvocationHandler로 인터페이스 기반 프록시를 만들고, CGLIB은 MethodInterceptor로 구체 클래스를 상속해 프록시를 만든다.
