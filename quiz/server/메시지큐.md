#### OX | [619]
Kafka는 컨슈머가 메시지를 받아가면 즉시 해당 메시지를 삭제한다.
> X
> Kafka는 메시지를 디스크에 로그 형태로 저장하며 컨슈머가 받아가도 삭제되지 않는다. 오프셋으로 위치를 관리해 재처리가 가능하다.

#### OX | [620]
RabbitMQ는 컨슈머가 ACK를 보내야 큐에서 메시지가 제거된다.
> O
> ACK 기반 신뢰성 전달 덕분에 컨슈머가 메시지를 성공적으로 처리했는지 브로커가 확인할 수 있다. 이 때문에 트랜잭션성 작업에 적합하다.

#### 빈칸 | [621]
Kafka에서 컨슈머가 마지막으로 처리한 위치를 나타내는 값은 ___ 이다.
> 오프셋
> 오프셋(offset)은 로그 내 메시지의 순서 번호다. 컨슈머는 마지막 오프셋을 기억해 중복 없이 다음 메시지를 요청할 수 있다.

#### 빈칸 | [622]
RabbitMQ가 사용하는 표준 메시지 프로토콜은 ___ 이다.
> AMQP
> AMQP(Advanced Message Queuing Protocol)는 메시지 브로커 표준 프로토콜이다. RabbitMQ 외에 ZeroMQ, ActiveMQ도 이를 구현한다.

#### 객관식 | [623]
RabbitMQ와 Kafka 중 대용량 스트리밍·클러스터링에 더 적합한 것은?
1. RabbitMQ — 메모리 저장으로 더 빠르다
2. ✅ Kafka — 디스크 로그 기반으로 클러스터링에 유리하고 배치 처리가 가능하다
3. 동일하다
4. 사용 목적에 무관하게 RabbitMQ가 항상 우수하다
> Kafka는 브로커를 여러 서버에 분산하는 클러스터링에 유리하고 배치 기능으로 대량 메시지를 효율적으로 처리한다.

#### 객관식 | [624]
트랜잭션성 작업에 더 적합한 메시지 큐는?
1. ✅ RabbitMQ — ACK 기반으로 신뢰성 있는 전달을 보장한다
2. Kafka — 오프셋 기반으로 재처리가 가능하다
3. 둘 다 동일하다
4. 메시지큐는 트랜잭션에 적합하지 않다
> RabbitMQ는 컨슈머 ACK로 메시지 처리 성공 여부를 브로커가 확인하므로 안정적인 처리가 필요한 트랜잭션성 작업에 적합하다.
