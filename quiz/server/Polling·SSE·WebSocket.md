#### OX | [643]
SSE는 클라이언트와 서버 간 양방향 통신을 지원한다.
> X
> SSE(Server-Sent Events)는 서버→클라이언트 단방향 통신이다. 양방향이 필요하면 WebSocket을 사용해야 한다.

#### OX | [644]
WebSocket은 HTTP 연결로 시작해 Upgrade 헤더로 프로토콜을 전환한다.
> O
> WebSocket은 초기에 HTTP로 핸드셰이크를 수행하고, Upgrade: websocket 헤더를 통해 TCP 기반 양방향 통신 프로토콜로 전환한다.

#### 빈칸 | [645]
SSE에서 서버가 끊긴 연결을 감지하기 위해 주기적으로 보내는 신호는 ___ 이다.
> heartbeat
> SSE는 서버가 연결 끊김을 즉시 파악할 수 없다. 주기적으로 heartbeat 메시지를 전송해 응답이 없는 연결을 감지하고 회수한다.

#### 빈칸 | [646]
클라이언트가 요청을 보내면 서버가 응답이 생길 때까지 연결을 유지하다가 응답 후 끊는 방식은 ___ 이다.
> 롱 폴링
> 롱 폴링은 일반 폴링과 달리 서버가 데이터가 있을 때까지 연결을 유지한다. 응답 후 재연결이 필요하고 커넥션을 점유하는 단점이 있다.

#### 객관식 | [647]
실시간 알림(주가, 뉴스 피드)처럼 서버→클라이언트 단방향 데이터 전송에 가장 적합한 방식은?
1. 폴링
2. 롱 폴링
3. ✅ SSE
4. WebSocket
> SSE는 HTTP 연결을 유지한 채 서버가 단방향으로 이벤트를 스트리밍한다. 채팅처럼 양방향이 필요하지 않은 경우 WebSocket보다 구현이 간단하다.

#### 객관식 | [648]
폴링의 단점으로 올바르지 않은 것은?
1. 불필요한 HTTP 요청으로 서버 부하 증가
2. 매번 TCP 연결을 맺고 끊어 오버헤드 발생
3. 실시간성 확보가 어렵다
4. ✅ 서버가 클라이언트에게 먼저 데이터를 푸시할 수 없다
> 폴링에서도 클라이언트가 요청을 보내면 서버가 최신 데이터를 응답으로 줄 수 있다. 폴링의 핵심 단점은 불필요한 반복 요청에 따른 오버헤드와 낮은 실시간성이다.
