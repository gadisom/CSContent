#### OX | [661]
Docker 컨테이너는 자체 OS 커널을 포함하고 있어 가상머신과 동일한 방식으로 동작한다.
> X
> 컨테이너는 호스트 OS 커널을 공유한다. 자체 커널이 없어 VM보다 가볍고 기동이 빠르다.

#### OX | [662]
같은 이미지로 여러 컨테이너를 동시에 실행할 수 있다.
> O
> 이미지는 불변 템플릿이므로 동일 이미지로 수십 개의 컨테이너를 독립적으로 생성·실행할 수 있다.

#### 빈칸 | [663]
Docker에서 컨테이너를 만들기 위한 불변 템플릿을 ___ 이라고 한다.
> 이미지
> 이미지는 컨테이너 생성의 기반이 되는 불변 파일이다. 레이어 구조로 구성되어 재사용성이 높으며 Docker Hub에서 공유한다.

#### 빈칸 | [664]
Docker가 "내 컴퓨터에서는 됩니다" 문제를 해결할 수 있는 이유는 컨테이너마다 ___ 이 격리되기 때문이다.
> 실행 환경
> Docker 컨테이너는 OS 라이브러리, 런타임 등 실행 환경을 이미지에 포함해 배포한다. 어느 호스트에서 실행해도 동일한 환경이 보장된다.

#### 객관식 | [665]
가상머신(VM)과 Docker 컨테이너의 차이로 올바른 것은?
1. VM은 커널을 공유하고 컨테이너는 자체 커널을 가진다
2. ✅ 컨테이너는 호스트 커널을 공유해 VM보다 가볍고 기동이 빠르다
3. 컨테이너는 보안상 VM보다 항상 우수하다
4. VM과 컨테이너는 동일한 방식으로 동작한다
> VM은 Hypervisor 위에 Guest OS(커널 포함)가 올라가 무겁다. 컨테이너는 호스트 커널을 공유해 OS 오버헤드 없이 가볍게 실행된다.

#### 객관식 | [666]
Docker 이미지와 컨테이너의 관계로 올바른 것은?
1. 이미지는 실행 중인 인스턴스, 컨테이너는 템플릿이다
2. 이미지와 컨테이너는 동일한 개념이다
3. 컨테이너를 먼저 생성하고 이미지를 나중에 만든다
4. ✅ 이미지는 불변 템플릿이고, 컨테이너는 이미지 위에 쓰기 레이어를 추가한 실행 인스턴스다
> 이미지 자체는 변경되지 않는다. 컨테이너 실행 시 이미지 레이어 위에 쓰기 가능한 레이어가 추가되어 런타임 변경이 격리된다.
